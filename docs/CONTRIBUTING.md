## How to Contribute

We welcome contributions of all kinds: code, documentation, bug reports, feature ideas, tests, and discussions. 
Here's how you can get involved:

## Steps to Contribute

1. **Fork the Repository**

   * Click the "Fork" button on GitHub to create your own copy of the repository.

2. **Create a Branch**

   * Always create a new branch from the `main` branch (or the default branch).
   * Use a descriptive name for your branch, e.g., `fix-bug-save-system`, `add-map-loader`, `improve-docs`.

3. **Make Changes**

   * Implement your feature, fix, or improvement.
   * Keep your changes focused and readable.

4. **Test Your Changes**

   * Ensure your code works as expected.
   * Run any existing tests and add new ones if applicable.

5. **Commit Your Work**

   * Write clear, descriptive commit messages.
   * Follow the style used in the project.

6. **Push and Open a Pull Request**

   * Push your branch to your forked repository.
   * Open a Pull Request (PR) against the main repository.
   * Describe your changes clearly and reference any related issues (e.g., `Closes #12`).

7. **Participate in Review**

   * Respond to feedback on your PR.
   * Make necessary updates and push them to your branch.
